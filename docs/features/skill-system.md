# Skill System

スキルの位置づけと agentmine の方針。

## 概要

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Skill System の方針                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  agentmine はスキルを管理しない。                                    │
│  各AIツールのネイティブ機能に委ねる。                                │
│                                                                     │
│  理由:                                                               │
│  - 各ツール（Claude Code, Codex等）が独自のスキル機構を持つ         │
│  - 形式・配置場所がツールごとに異なる                                │
│  - 統一管理は現実的でない                                           │
│                                                                     │
│  agentmine の責務 = タスク管理                                       │
│  スキル管理 = 範囲外                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## スキルとは

再利用可能なプロンプトテンプレート。AIエージェントに「やり方」を教える。

```
スキル = 知識・ガイダンス（Guidance）
ツール = 能力・権限（Capability）

スキルは「許可」ではなく「指示」
権限制御はツール・サンドボックス層で行う
```

## agentmine での扱い

### エージェント定義

エージェントにスキルを紐づける概念は **廃止**。

```yaml
# Before (廃止)
agents:
  coder:
    skills: [implement, test]  # ← これは使わない

# After
agents:
  coder:
    model: claude-sonnet
    tools: [Read, Write, Edit, Bash]
    # スキルは各ツールのネイティブ機能で
```

### スキルの利用

エージェントが必要に応じて、各ツールのネイティブ機能でスキルを読み込む。

```
Agent (Claude Code) → .claude/commands/implement.md を Read
Agent (Codex) → Codex 独自の方法でスキル取得
```

agentmine は関与しない。

## 設計上の決定事項

| 項目 | 決定 |
|------|------|
| スキル管理 | agentmine の範囲外 |
| スキル合成 | しない（独立した知識単位） |
| バリデーション | 各ツールに委ねる |
| エラー処理 | 各ツールに委ねる |
| 権限制御 | スキルではなくツール層で |

## 将来の検討事項

各ツール間でスキルを共有したい場合:

1. 共通フォーマットの策定
2. ツールごとのアダプター実装
3. マーケットプレイス

現時点では **実装しない**。需要が明確になってから検討。
